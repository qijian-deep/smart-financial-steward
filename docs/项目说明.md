# 智能财务规划助手 — 项目说明文档

> 本文档依据当前仓库源码整理，描述产品功能、技术实现与运维要点。若与根目录 `README.md` 存在出入，**以本文件及源码为准**。

---

## 1. 项目定位与价值

**智能财务规划助手**是一款在浏览器中运行的单页应用（SPA），面向有稳定收入、希望用「假设—推演」方式理解资产配置后果的用户。

- **产品边界**：用历史基金净值等数据做**回测与情景展示**，帮助用户看清参数变化下的资产曲线与风险指标；**不是**投顾平台，不提供买卖指令。
- **典型用法**：输入基金代码加载历史净值 → 配置分段收入、支出、多基金定投、存款与初始资金 → 选择模拟区间 → 运行计算 → 查看曲线、收益、回撤、IRR 与风险提示。

## 2. 功能说明

### 2.1 基金数据

| 能力 | 说明 |
|------|------|
| 按代码加载 | 用户输入基金代码，向东方财富基金页请求 `pingzhongdata/{code}.js`，解析全局变量中的净值序列。 |
| 本地缓存 | 成功加载的代码写入 `localStorage`；下次进入页面会批量重新拉取脚本并刷新列表（失败代码会从列表中剔除）。 |
| 数据处理 | 原始点按「年-月」聚合，计算月初/月末净值、月度增长率、累计分红等，供模拟引擎按月引用。 |

### 2.2 输入侧（`InputSection`）

- **基金**：多笔定投配置（代码、每月金额、起止年月）；可选 **止盈率（%）**、**止盈后重投入分期（月）**；支持将当前多基金配置保存为**预设**并写入 `localStorage` 以便复用。
- **收入**：多段「月收入 + 生效起止年月」，引擎按月份匹配落在哪一段（重叠段会加总）。
- **支出**：统一月支出；**年额外支出**为若干金额列表，在引擎内按 **全年总额 ÷ 12** 摊到每月（与「每年一次性扣款」不同，实现上为月均负担）。
- **存款**：多笔；每笔含金额、起止年月、**年利率**；在存续月份按 `年利率/12` 计算当月利息并计入收益（见引擎实现）。
- **现金**：**初始余额**，作为首月可支配资金的起点，并参与后续「累计可支配收入」滚动。
- **模拟区间**：起止年月（`YYYY-MM`），与基金定投区间、收入区间等共同约束有效月份。
- **未来视图**：可配置 **平移年数**，仅用于展示层把历史结果的时间轴整体后移（金额不变）。
- **操作**：拉取基金、**运行模拟**、**重置缓存**（清空多类 `localStorage` 并恢复引擎默认状态）。

### 2.3 输出侧（`OutputSection`）

- **历史 / 未来** 双视图切换；未来视图为客户端对 `SimulationResult.monthlyData` 的月份平移。
- **折线图**（AntV G2Plot）：**组合总资产**与**累计投入**（纵轴以「万元」展示）；最大回撤点用标注点标出。
- **统计卡片**（历史视图）：期末资产、总投入、总收益（额/百分比）、年化收益、**IRR（内部收益率）**、最大回撤（比例与发生月份）、最大亏损金额。
- **风险提示**：例如最大回撤比例过高、最大回撤金额相对当月参考收入的倍数等（部分历史文案在源码中已注释，以当前 `OutputSection` 为准）。
- **免责声明**：固定展示非投资建议声明。

### 2.4 模拟引擎中的进阶行为（止盈与债券基金）

- 当某基金配置 **止盈率** 且当月收益率（相对累计成本）达到阈值时，逻辑上对该基金做 **全额赎回**，成本清零。
- 若配置了 **重投入分期**，赎回资金会在后续月份以 **额外定投** 形式加回该基金。
- 若组合中存在 **名称包含「债券」的基金**，止盈赎回款会进入 **第一支债券基金** 的虚拟仓位，并按该债基当月净值增长率滚动；若无债基，止盈赎回款进入 **按 1.5% 年化计息** 的现金池（在引擎内以月利息形式并入分红/收益累计项）。

以上规则均以 `SimulationEngine` 源码为准，属于简化的行为模型，与真实交易费用、申赎规则、Tax 等无关。

## 4. 架构与目录职责

```
src/
├── main.tsx                 # 挂载 React 根节点
├── App.tsx                  # 编排：持久化、订阅引擎/加载器、重置、子组件 props
├── App.css / index.css      # 应用布局与全局主题变量（含深色偏好）
├── types/index.ts           # 领域类型（基金、配置、月度结果、SimulationParams 等）
├── services/
│   ├── FundDataLoader.ts    # 脚本注入拉数、处理、事件 `allFundsLoaded`
│   ├── FundStorage.ts       # 基金代码列表 localStorage
│   └── SimulationEngine.ts  # 状态 + calculate() + 事件 `resultChange`
├── utils/fundDataProcessor.ts  # 净值分组、分红与增长率衍生计算
└── components/
    ├── InputSection.tsx     # 左侧表单与业务控件
    └── OutputSection.tsx    # 右侧图表与指标卡片
```

**设计特点**：

- **服务单例**：`fundDataLoader`、`simulationEngine` 导出单例，便于在非 React 模块间共享；`App` 中通过 `subscribe` 同步 UI。
- **计算与 UI 分离**：核心财务逻辑集中在 `SimulationEngine`，组件只负责采集参数与展示。
- **无后端**：数据来自第三方脚本；用户配置落在浏览器 `localStorage`。

更细的「按文件」速查可配合 `src/README.md` 使用。

## 5. 数据流概览

1. 用户输入基金代码 → `FundDataLoader.loadFundData` 插入 `<script src="https://fund.eastmoney.com/pingzhongdata/{code}.js">`。
2. `onload` 后读取 `window.Data_netWorthTrend`、`window.fS_name` → `processFundData` → 写入内存中的 `LoadedFundData[]`。
3. `App` 将表单数据写入 `simulationEngine` 各 setter，并序列化到 `localStorage`（见下节）。
4. 用户点击计算 → `simulationEngine.calculate()` → 生成 `SimulationResult` → `resultChange` → `OutputSection` 更新图表与卡片。

开发调试：`App` 在 `useEffect` 中会把 `fundDataLoader`、`simulationEngine` 挂到 `window`，便于在控制台探查。

## 7. 核心计算要点（`SimulationEngine.calculate`）

以下为阅读源码时的摘要，**不等同于金融或会计定义**，实现细节以代码为准。

1. **月份序列**：在 `mockStartDate`～`mockEndDate` 间按自然月枚举。
2. **可支配收入**：每月 `月收入 - 月支出 - 月均年额外支出`；首月加上 `initialBalance` 得到「累计可支配收入」轨迹。
3. **定投**：对每个在投月份的基金，累加基础定投与止盈重投入分摊额，得到当月总定投 `monthFundInvestment`。
4. **累计投入字段**：当前实现中 `monthlyData[].cumulativeInvestment` 与 **累计可支配收入** 对齐（用于与总资产对比及 IRR 现金流构造），与「仅统计买入基金的本金」的直觉可能不同，阅读报表时需注意。
5. **基金市值**：按月初资产 + 当月实际投入后乘以该月净值**增长率**（缺月时向前查找最多 12 个月）；叠加止盈、债基池、分红等分支。
6. **存款收益**：存续月内 `金额 × (年利率/100/12)` 求和进入 `depositIncome`（在总资产公式中与现金、基金、分红累计等合并）。
7. **总资产**：普通基金市值 + 债券基金池 + 现金（累计投入 − 累计定投）+ 存款收益累计项 + 分红等累计项 + 无债基时止盈现金池及其利息的处理（见源码注释）。
8. **总收益 / 年化**：由首尾月的 `cumulativeInvestment` 与 `totalAssets` 推导；年化为简单复利形式的时间加权近似。
9. **IRR**：由每月「累计投入增量」与最后一期终值构造现金流，**二分法**求月 IRR 再年化；若无解则回退为年化收益率。
10. **最大回撤**：在总资产序列上跟踪历史峰值，记录最大回落金额与比例及对应月份。

## 8. 构建、部署与工程配置

- **开发**：`npm run dev` — Vite 开发服务器默认 **5173**，`vite.config.ts` 中 `server.host` 为 `0.0.0.0` 便于局域网访问。
- **生产 base**：`vite.config.ts` 中 `base: '/smart-financial-steward/'`，用于 **GitHub Pages** 子路径部署；若部署在域名根路径，需按需修改 `base` 与托管平台路由。
- **Vercel**：仓库内 `vercel.json` 使用 `@vercel/static-build`，`distDir: dist`，SPA 回退到 `index.html`。
- **外网穿透**：`npm run tunnel` 使用 ngrok 暴露本地 5173（依赖 `npx ngrok` / 本地 ngrok 配置）。
